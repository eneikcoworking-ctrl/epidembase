# Стратегия миграции Базы Знаний в Moodle (sdo.crie.ru)

## 1. Сопоставление объектов
Текущая база знаний на лендинге проектируется с учетом будущей структуры Moodle:

| Элемент лендинга | Ресурс/Элемент Moodle | Примечание |
| :--- | :--- | :--- |
| Статья (Article) | Ресурс «Страница» | Подходит для инструкций, FAQ и методик. |
| Документ (Document) | Ресурс «Файл» | Для PDF, DOCX, XLSX шаблонов и регламентов. |
| Группировка статей | Элемент «Книга» | Для структурированных руководств с главами. |
| Классификатор (Категории) | Категории курсов / Разделы | Иерархическая структура по направлениям. |
| Глоссарий | Элемент «Глоссарий» | Для списка сокращений и терминов. |
| Метаданные (Теги) | Теги Moodle | Для сквозного поиска. |

## 2. Перенос данных
- **Метаданные:** Поля `status`, `version`, `responsible` будут перенесены в описание ресурсов или пользовательские поля метаданных.
- **Видео:** Ссылки на видео-инструкции интегрируются через медиа-вставки в «Страницы» или «Книги».
- **Телесуфлеры:** Тексты телесуфлеров сохраняются как скрытые разделы или дополнительные поля описания.

## 3. Авторизация и Доступ
- Использовать существующую систему авторизации ЭИОС (LDAP/OIDC).
- Синхронизировать роли:
    - Администратор БЗ -> Управляющий категорией.
    - Контент-менеджер -> Учитель (без права выставления оценок).
    - Обучающийся -> Студент.

## 4. Этапы миграции
1. Экспорт данных из текущей системы в JSON/CSV.
2. Создание структуры категорий в Moodle.
3. Пакетная загрузка файлов и создание страниц через API Moodle или плагины импорта.
4. Настройка прав доступа и уведомлений.
